# Contributing to Crateria projects

1. Open an issue first for large changes.
2. Fork or branch from `main`.
3. Keep commits focused; run the repo’s CI locally when practical (`cargo test`, `cargo clippy`, `cargo deny` where configured).
4. Prefer the canonical package-repo install docs in product READMEs.
5. Do not commit secrets, signing keys, or large binary pools outside the `packages` release workflow.
